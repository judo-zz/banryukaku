# 蟠龍閣 ARG サイト — 他AIへの相談用まとめ

> 制作中のARG（Alternate Reality Game）サイトについて相談したい。以下の概要を読んで提案・意見をください。

---

## 1. コンセプト

**一行定義：**
「ただのコンカフェサイトを調べているだけ……なのに、君はもう龍牌会の『次の龍』に育っていた。」

**ジャンル：** 静的HTML+CSS+JSのみで動くARGサイト（GitHub Pages）

**入り口：** Xの怪しい投稿から流入。「蟠龍閣の検索窓にゆめって入れたら……」という告知投稿がプロローグ。

**エンド：** プレイヤーが龍牌会のサイトを「乗っ取った（新・教祖になった）」という完全逆転エンディング。localStorageで永続化。

---

## 2. 世界観・ストーリー

**表の顔：** 秋葉原のネオ・オリエンタル コンカフェ「蟠龍閣（ばんりゅうかく）」。賑やか・華やか・中華ネオン。

**裏の顔：** 地下組織「龍牌会（ろんぱいかい）」が運営する、外部から「次代の龍（新教祖）」を育てる装置。

**事件：** キャスト「ゆめ」が「失敗作」と判定され、2024年11月3日に地下施設で「廃棄」された。彼女は廃棄直前にサイト内へバックドア（隠しページ群）を仕込んだ。

**プレイヤーの役割：** 失踪したゆめを心配して調べ始めた一般人。調べるほどサイトが変化し、気づけば龍牌会に「次の龍」として認識されている。

---

## 3. investigationLevel システム

`localStorage`（`br_level`・`br_flags`）で管理。検索キーワードヒット時に自動上昇。

| Level | 到達条件 | サイトの変化 |
|---|---|---|
| 0 | 初回訪問 | 完璧に美しい本物コンカフェ |
| 1 | 「ゆめ」検索（初回） | ネオンが微かにチラつく。faviconが金色に |
| 2 | 隠しページ2つ発見 | 背景に龍鱗グリッチ。タブタイトル変化 |
| 3 | Lv2以上のキーワード発見 | 金色が全体に侵食 |
| 4 | RENPAI解除 | 管理者権限移行中アラート |
| 5 | 最終ページ到達 | 完全黒金化。「君の蟠龍閣」に永続化 |

---

## 4. 攻略導線（SEARCH_INDEX 実装済み）

サイト内検索ボックス（全ページ共通）にキーワードを入力するとページ遷移する。ヒントはHTMLコメントではなくすべて本文テキストに自然に埋め込まれている。

```
検索ワード          → 遷移先ページ                    到達Lv
─────────────────────────────────────────────────────────
「ゆめ」「夢」      → hidden/yume-notice.html          Lv1
「YM-2023-04」      → hidden/yume-rireki.html           Lv2
「黒瀬」            → hidden/kurose-shiji.html          Lv2
「失敗作」          → hidden/yume-finallog.html         Lv2
「龍牌会」          → hidden/ronpaikai-chart.html       Lv2
「MAP-RY-023」      → hidden/basement-map.html          Lv3
「廃棄」            → hidden/disposal-record.html       Lv3
「選定」            → hidden/selection-criteria.html    Lv3
「RENPAI」          → hidden/backdoor.html              Lv4  ← 山場
「龍牌会 端末」     → hidden/admin-console.html         Lv4  ← 2ワード山場
```

**前提レベルチェック：** `level - 1` 未満のプレイヤーは検索しても「該当するデータが見つかりません」とだけ返す（先読み防止）。

## 5. キャラクター設定

### ゆめ（不在・廃棄済み）
- 最終出勤：2024.11.03
- 同期：ちゃき（9/15〜）
- 性格：のんびり・観察眼がある
- ARGでの役割：バックドアを仕込んだ人物。プレイヤーへのメッセンジャー
- ブログ：全11記事（2024.09.01〜2024.11.02）

**ゆめブログ最終記事（2024.11.02「たいせつなこと」）縦読みギミック：**
```
れんらく、したかったけど、うまくできなかった。
んー、でもこれだけは残しておきたかった。
ぱっと見は普通の日記に見えると思う。
いつかここに来た人への、手紙だよ。

縦読み → れ・ん・ぱ・い = RENPAI（検索キーワード）
```

### ちゃき
- ゆめの同期。ゲーム・お絵かきが趣味
- ARGヒント：「黒瀬係長から連絡が増えた気がする」
- ゆめ失踪後も積極的にゆめブログを読み返している（なのブログへの誘導役）

### なの
- カフェ・読書が趣味
- ARGヒント：「龍牌会って言葉、どこかで聞いたことがある」（読んだ小説「恋敗」に出てきた）
- ミステリ好き設定がゆめブログの謎読みへの橋渡しになっている

### みじゅ
- 料理・映画が趣味。先輩ポジション
- ARGヒント：「先代オーナーが『外から来る者が四代目』と言っていた」

### よすず
- 音楽・占いが趣味
- ARGヒント：「あなたがここを読んでいるなら、もう選ばれている」

### 黒瀬（係長）
- 龍牌会の実務担当。廃棄命令を出した人物
- 「悪人ではなくシステムに従う人間」として描写。感情の気配を少し残している
- ゆめブログに名前が登場（係長に声をかけられた記事）→「黒瀬」検索に誘導

## 7. 文体・デザイン方針

**文体（隠しページ）：**
- 森博嗣スタイル：感情表現なし。行動・事実・記録のみ。タイムスタンプ必須
- 西尾維新スタイル：N番目構造（#001→#002→#003の連鎖）。未解決エンディング

**禁止ワード（絶対使わない）：** 諦めない・ありがとう・覚えていてください・遺志

**ARG没入感ルール：** HTMLに「（架空）」「フィクション」などの注釈を入れない。プレイヤーには現実として見せる。

**デザイン：**
- 表ページ：深紅・金・ネオンブルー。賑やかで本物っぽい
- 隠しページ：黒背景・monospace端末デザイン
- investigationLevelで段階的に侵食（CSS `data-level` 属性で制御）

## 8. 現在の課題・相談したいこと

### 課題A：Lv1-2の「ゆめを探す」体験が薄い

**現状の問題：**
ブログ（Lv0）でゆめに感情移入した後、すぐに「機関の公文書」（yume-notice → 在籍記録 → 黒瀬メール）に入る。
「失踪ミステリーとして始まり、調べていくうちに龍になっていく」という流れを実現したいが、現状はLv1-2で既に組織の記録を読んでいるため、「探している感」より「記録を読んでいる感」になっている。

**案：** ゆめの個人的な痕跡（未送信メモ、下書きメッセージ等）を1ページ追加し、「私的な声→機関の冷たい記録」という温度差を作る。

→ この案の是非・代替案を聞きたい。

---

### 課題B：Lv3の緊張感が弱い

プレイヤーが「自分も廃棄されるかもしれない」と感じる演出がLv3にない。
選定基準・廃棄記録を読んでいるのに、あくまで「観客」として読んでいる印象。

→ プレイヤーを「観察される側」にする演出アイデアがほしい。

---

### 課題C：Lv5の反転感が弱い

「ゆめを探しに来たのに龍になった」という気づきの一文が最終ページに足りていない。
エンディングが「権限取得成功」の達成感に偏っていて、「自分がそうなっていたことへの気づき」がない。

→ この落差をどう演出するか意見がほしい。

---

### 課題D：他ブログへの画像追加

現在画像があるのはちゃきブログ（お出かけ写真）のみ。
- みじゅ：料理写真
- なの：カフェ写真  
- よすず：タロット
- ゆめ：パフェ（記事はあるが画像なし）

→ どのブログ・どの記事に画像があると自然か、優先度の観点で整理してほしい。

## 9. 技術制約

- 静的HTML+CSS+JS のみ（バックエンドなし）
- GitHub Pages で動作（judo-zz.github.io/banryukaku）
- investigationLevel は `arg.js` で一元管理（localStorage）
- `robots.txt` で /hidden/ をdisallow済み
- フォントは `public-fonts.css` で Google Fonts 読み込み（表ページのみ）
